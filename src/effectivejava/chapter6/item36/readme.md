# 아이템 37. ordinal 인덱싱 대신에 EnumMap을 사용하라.

---
## 과거의 기법
- 비트 필드를 활용해서 집합 상태를 정리
- 예 > 구매 프로세스
- 1 : 체결			(1 << 0)
- 2 : 세금계산서	(1 << 1)
- 4 : 물대지급		(1 << 2)
- 8 : 국세청		(1 << 3)
- 15가 모든 것이 끝난 것
- 15가 아니면 뭐가 모자라도 모자라는 것

---
## 과거의 기법의 단점
- 14 : 세금계산서 미발행
- 15 : 미체결
- 11 : 대금지급지연
- 07 : 국세청 전송오류
- 이걸 다 외워야 하는 단점 ㅠㅠ
- 내가 이러려고 개발자가 되는지 자괴감이 들어 

---
## 핵심 정리
- 열거 타입을 집합 형태로 사용한다고 해도 비트 필드를 사용하지 말고 EnumSet 클래스를 사용하자.
