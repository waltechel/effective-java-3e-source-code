# 아이템 29. 이왕이면 제네릭 타입으로 만든다.

---
## E[] 로 배열을 바꾸는 방법 vs Object[] 배열을 사용하되 배열을 읽을 때마다 E 로 형변환하는 방법
- E[]로 배열을 바꾸는 방법
	- 가독성이 좋다.
	- 한 번만(멤버 필드 정도에서) 바꾸면 끝난다.
	- 배열의 런타임 타입이 컴파일 타입과 달라 문제가 생길 수 있다.
- 읽을 때마다 E로 바꾸는 방법
	- 가독성은 좀 별로이다.
	- 배열을 가지고 보는 것은 아니다.

---
## 핵심 정리
- 제네릭 타입이 안전하고 쓰기 편하다.
- 새로운 클래스를 설계할 때는 형변환 없이도 사용할 수 있도록 변경한다.
